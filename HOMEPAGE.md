## 0. 定位与整体思路

### 0.1 核心定位（一句话）

> **一个自托管的 AI 统一网关 —— 一套标准 API 接入海量大模型，内置对话 / 图像 / 视频体验中心。**

### 0.2 相比现状的关键调整

**严格保持现有 5 区块结构不变**（Hero → Stats → Features → HowItWorks → CTA），不新增区块、不展示体验中心。本方案只在现有骨架内做内容校准与视觉升级：

- **保留**现有 5 个区块的骨架与素白极简视觉，区块顺序、数量完全不动。
- **校准**统计数字与卖点文案，使其贴合项目真实能力，避免夸大。
- **视觉**统一到 DESIGN.md 素白 Monochrome 规范（黑墨重粗标题、蓝为点睛、1px 细线卡片）。

### 0.3 目标受众

主要面向**开发者 / 团队技术负责人 / 自托管运维者**（toB、专业、冷静）。文案克制、以能力和数据说话，不用营销腔。

## 3. 移动端适配总原则（横切）

1. **断点**：沿用现有 `md:`（≥768px）为主分界；`lg:`（≥1024px）用于 Hero 双列。≤768px 一律单列。
2. **Hero**：终端 Demo 降到内容下方，主 CTA 占满宽度，装饰性渐变/网格背景降低不透明度或隐藏以省性能。
3. **触控目标**：所有按钮/链接最小高度 44px（DESIGN.md Button Large=48px 优先）。
4. **横向滚动**：模型 pill、支持应用列表在窄屏用横向滚动容器，避免挤压换行错乱。
5. **图片/预览**：新增体验中心区块的预览图务必懒加载 + 固定宽高比，防止 CLS 抖动。
6. **字号**：主标题用 `clamp()`（现状已用），保证小屏不溢出。
7. **留白**：区块纵向 padding 在移动端从 `py-24/32` 收到 `py-16` 左右，避免过空。
8. **动画**：尊重 `prefers-reduced-motion`（现状 Stats/Counter 已处理，其他区块同样遵循）。
